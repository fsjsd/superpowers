## Superpowers Script Protocol

This script will be run inside an Electron app "Super Powers" . It must conform to the following protocol exactly.

If it is not clear how the script should function with this protocol, ask me questions until you understand the requirement.

---

### 1. Describe mode

When the script is invoked with the describe flag, it must print a single JSON descriptor to stdout and exit with code 0. No other output should be printed.

**Node / TypeScript:**

```
node script.js --superpowers=describe
npx tsx script.ts --superpowers=describe
```

**Python:**

```
python script.py --superpowers describe
```

The JSON must match this shape:

```json
{
  "name": "Human-readable script name",
  "description": "One-sentence description of what the script does",
  "category": "Optional category label, e.g. Media / Data / Files",
  "requirements": "Optional human-readable requirements, e.g. Node v20+, ffmpeg installed",
  "author": "Optional author name",
  "icon": "Optional name of a lucide.dev icon for the script",
  "input_schema": [
    {
      "name": "param-name",
      "type": "folderpath | filepath | text | boolean | number | select",
      "label": "Human-readable label",
      "description": "Optional helper text shown under the field",
      "required": true,
      "default": "",
      "options": ["only for select type"]
    }
  ],
  "events": [
    {
      "type": "progress",
      "payload_schema": [
        { "name": "total", "label": "Total items", "type": "number" },
        { "name": "finished", "label": "Items completed", "type": "number" }
      ]
    }
  ],
  "output_schema": [
    {
      "type": "csv_file | media | html",
      "label": "Human-readable output label"
    }
  ]
}
```

---

### 2. Input arguments (run mode)

In run mode the app passes inputs as CLI arguments in the form `--name=value`, matching the `name` fields in `input_schema`. Parse them accordingly.

---

### 3. Stdout during a run

All structured events must be emitted as **newline-delimited JSON** on stdout. Any stdout line that is not valid JSON is treated as a plain log message and shown in the log panel.

#### Progress event

Emit periodically during work. The payload field names must match the `payload_schema` defined in `events`:

```json
{ "event": "progress", "payload": { "total": 100, "finished": 42, "errors": 0 } }
```

#### Output event

Emit **once** when the output file is ready. This tells the app where the result is and what type it is:

```json
{ "event": "output", "payload": { "path": "/absolute/path/to/output.csv", "type": "csv_file" } }
```

`type` must be one of: `csv_file`, `media`, `html`

---

### 4. Exit codes

- Exit `0` on success.
- Exit non-zero on failure. Log a descriptive error message to stderr before exiting.

---

### 5. Input type notes

- `folderpath` — path to a directory
- `filepath` — path to a single file
- `text` — plain string
- `boolean` — pass `--flag=true` or `--flag=false`; parse the string value
- `number` — numeric string; parse with parseInt / parseFloat
- `select` — one of the values listed in `options`

---

### 6. Output type notes

- `csv_file` — write a CSV file to disk and emit an `output` event with its absolute path. Include a header row.
- `media` — write an image or video file and emit its absolute path.
- `html` — write an HTML file and emit its absolute path. The app will sanitise it with DOMPurify before rendering.

---

### 7. Describe mode detection (implementation pattern)

**Node / TypeScript:**

```js
const args = process.argv.slice(2);
if (args.includes('--superpowers=describe')) {
  console.log(JSON.stringify(descriptor));
  process.exit(0);
}
```

**Python:**

```python
import sys, json
if '--superpowers' in sys.argv and sys.argv[sys.argv.index('--superpowers') + 1] == 'describe':
    print(json.dumps(descriptor))
    sys.exit(0)
```
